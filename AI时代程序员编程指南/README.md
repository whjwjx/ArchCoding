## 🔑 核心观点

### 人机协作的本质

人抓"顶层决策、架构方向、规范标准、风险边界、业务逻辑"，AI干"细节落地、代码编写、文档产出、重复劳动、问题排查"。

### 五大铁律

1. **需求拆解原则**：需求不拆细，绝不让AI动手
2. **规范先行原则**：规范先行，开发后置
3. **任务拆分原则**：一次只给AI一个小任务
4. **架构控制原则**：架构&技术栈人全权决定，AI无权选型
5. **人工审核原则**：AI所有产出，人工必审核兜底

### 最优工作流

1. 批量生成：架构+目录+公共文件+数据库模型
2. 批量生成：所有通用CRUD基础接口
3. 逐个开发：复杂业务个性化接口
4. 最后统一让AI：格式化代码、补注释、整理文档
